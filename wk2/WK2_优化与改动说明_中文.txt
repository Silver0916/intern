=====================================
一、整体改动概览
=====================================
本轮针对 wk2 的主要目标：
1. 修复原有 VO 主流程中的结构性错误（接口不一致、变量未定义、GT 读取不兼容）。
2. 让代码可直接跑 EuRoC 数据（cam0/sensor.yaml + state_groundtruth_estimate0/data.csv）。
3. 增强可视化评估公平性（加入 Sim(3) 对齐）。
4. 增强鲁棒性（去畸变、低视差过滤、参数可调、隔帧处理）。
5. 完善输出文件（过程 CSV 与映射 CSV）。

涉及文件：
- wk2/vo_monocular.py
- wk2/run_task2.py
- wk2/visualize_trajectory.py


=====================================
二、wk2/vo_monocular.py 的改动与优化
=====================================
1) 主流程接口统一
- 统一为：run_monocular_vo(frames_dir, intrinsics_yaml, gt_csv=None, ...)
- 从“单对帧临时逻辑”恢复为“整段序列连续处理”。

2) 相机内参读取增强
- 支持以下 YAML 格式：
  a. K (3x3)
  b. K (长度9一维数组)
  c. intrinsics: [fu, fv, cu, cv]（EuRoC/Kalibr常见）
  d. fx/fy/cx/cy
- 增加异常提示，避免形状错误静默失败。

3) 畸变参数读取与使用（新增）
- 新增 _read_distortion_coeffs：读取 distortion_coefficients 或 D。
- 若有畸变参数，先用 cv2.undistortPoints 归一化点，再估计 E、恢复位姿。
- 统计字段新增 distortion_compensated（是否启用去畸变）。

4) GT 读取与对齐增强
- 支持两类 GT：
  a. frame,tx,ty,tz 格式
  b. timestamp + p_RS_R_x/y/z（EuRoC 原始）
- 时间戳对齐从“最近邻”升级为“线性插值”，减少离散跳变误差。
- 新增输出 gt_poses_aligned.csv（按 frame 对齐后的 GT，方便可视化与评估）。

5) 几何估计鲁棒性增强
- recoverPose 使用 findEssentialMat 的 e_mask。
- 增加 recoverPose mask 有效性检查。
- 新增最小视差门限 min_parallax_px：
  - 计算 pose inlier 的光流中位数
  - 低于阈值则拒绝更新（避免低基线/退化帧导致错误位姿累积）。

6) 隔帧处理能力（新增）
- 新增 frame_step 参数，支持每 N 帧取一帧进行 VO。
- 统计中新增：num_frames_total、frame_step、num_frames（采样后）。

7) CSV 输出增强
- estimated_poses.csv
- vo_pair_metrics.csv
- metrics_summary.csv
- frame_index.csv（frame -> filename -> timestamp）
- gt_poses_aligned.csv（新增）
- 在轨迹与pair指标中新增 median_parallax_px 字段。


=====================================
三、wk2/run_task2.py 的改动与优化
=====================================
1) 默认流程调整
- 默认不再依赖仿真数据生成。
- 直接使用外部输入：frames_dir / gt_csv / intrinsics_yaml。

2) 仿真模式改为显式触发
- 通过 --use_sim 启用仿真。
- 仅在 --use_sim 时尝试导入 simulation_drone_sequence。

3) 动态导入优化
- 使用 importlib.import_module("simulation_drone_sequence")。
- 目的：避免 IDE 对缺失模块的静态导入报错干扰默认数据集流程。

4) 参数透传增强
- 新增并透传：min_parallax_px、frame_step。
- 默认参数调整为更稳健：ratio_test=0.75、min_inliers=20。

5) 可视化 GT 选择修复
- 优先使用 vo_result["aligned_gt_csv"] 进行可视化。
- 使 EuRoC 原始 timestamp GT 能直接进入可视化流程，无需手工转换。


=====================================
四、wk2/visualize_trajectory.py 的改动与优化
=====================================
1) 新增 Sim(3) 对齐
- 新增 _align_sim3（Umeyama），在绘图前将 Estimated 轨迹对齐到 GT。
- 作用：消除全局旋转、平移、尺度差异，使比较更公平。

2) 误差计算逻辑更新
- 误差从“直接点差”改为“对齐后点差”。
- 图标题明确标注 Sim(3) Aligned，避免误解评估口径。


=====================================
五、当前版本 vo_monocular.py 函数功能解析
=====================================
以下为当前函数职责说明：

1. _read_intrinsics(path)
- 读取相机内参并返回 3x3 K。
- 兼容 K / intrinsics[fu,fv,cu,cv] / fx,fy,cx,cy。

2. _read_distortion_coeffs(path)
- 读取畸变参数（distortion_coefficients 或 D）。
- 返回列向量；无参数时返回 None。

3. _orthonormalize_rotation(r)
- 使用 SVD 将近似旋转矩阵投影回 SO(3)。
- 防止累计误差导致旋转矩阵失真。

4. _write_dict_rows(path, fieldnames, rows)
- 通用 CSV 写出函数。

5. _safe_mean(values)
- 空列表返回 NaN；否则返回均值。

6. _strip_row_keys(row)
- 清理 CSV 列名左右空格，兼容不同导出格式。

7. _get_first_key(row, candidates)
- 在候选列名中取第一个有效值。
- 用于兼容不同数据源列名差异。

8. _load_gt_positions_for_frames(gt_csv, frame_paths)
- 读取 GT 并按 frame index 返回位置字典。
- 支持 frame 格式与 timestamp 格式。
- timestamp 格式下按帧名时间戳进行线性插值对齐。

9. _write_frame_index_csv(path, frame_paths)
- 输出 frame 与 filename/timestamp 对照表。
- 便于调试和追溯问题帧。

10. _write_aligned_gt_csv(path, gt_positions)
- 将对齐后的 GT 统一导出为 frame,tx,ty,tz。
- 供可视化和后续评估使用。

11. run_monocular_vo(...)
- 核心主流程：
  a. 读取输入与初始化状态
  b. ORB 特征提取 + KNN + ratio test
  c. Essential Matrix + RANSAC
  d. recoverPose 恢复相对位姿
  e. 视差门限筛选、尺度恢复（GT步长或默认1）
  f. 累积全局位姿
  g. 输出多类 CSV 与统计指标

12. _build_arg_parser()
- vo_monocular.py 独立运行时的命令行参数定义。


=====================================
六、当前参数建议（EuRoC MH_01_easy）
=====================================
建议起步参数：
- ratio_test: 0.75
- min_inliers: 20
- min_parallax_px: 1.0
- frame_step: 1

若出现大量失败帧，可尝试：
- ratio_test: 0.7
- min_inliers: 30
- min_parallax_px: 1.5
- frame_step: 2

注意：
- 目前仍是“前端主导”的单目 VO，长序列漂移难以完全避免。
- 若要显著缩小轨迹偏差，建议引入关键帧 + PnP + 局部 BA（或直接采用成熟 SLAM/VIO 框架）。
